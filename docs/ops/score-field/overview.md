# 记分场运营概览（官方）

> 记分场由 **平台官方** 运营；**各游戏使用独立游戏金币**结算。  
> 房卡场（地推）见 [room-card/overview.md](../room-card/overview.md)。

---

## 1. 定位

| 项目 | 说明 |
| :--- | :--- |
| **运营主体** | 平台官方运营团队 |
| **场域性质** | 陌生人快速匹配、竞技娱乐 |
| **变现方式** | 平台抽水（5%）、可选桌费 |
| **目标用户** | 有竞技需求的活跃用户 |
| **结算货币** | 各游戏独立金币（如打乌龟「龟币」） |

游戏金币参数见 [games/{id}/ops-hooks.md](../../games/README.md)；抽水框架见 [tiers-rake.md](tiers-rake.md)。

---

## 2. 上线节奏

**前置条件（Phase 1 房卡场验证通过后）：**

- 房卡场月活 ≥ 5,000
- 7 日留存 ≥ 30%
- 房卡复购率 ≥ 40%

**上线时间：** Phase 1 完成后 3~6 月；可按游戏逐个开放记分场。

---

## 3. 官方运营职责

| 职责 | 说明 |
| :--- | :--- |
| 场次管理 | 按游戏配置准入与底分（见各游戏 ops-hooks） |
| 匹配调度 | 按**游戏维度**隔离匹配；防作弊同桌 |
| 抽水结算 | 赢家净赢 × 5%，详见 [tiers-rake.md](tiers-rake.md) |
| 活动运营 | 签到、锦标赛等，奖励以当前游戏金币发放 |
| 破产保护 | 见 [matching-bankruptcy.md](matching-bankruptcy.md) |

---

## 4. 双轨并行规则（记分场 vs 房卡场）

| 规则 | 房卡场 | 记分场 |
| :--- | :--- | :--- |
| 运营主体 | 地推 / 代理 | 平台官方 |
| 房卡 | 消耗平台房卡（**全游戏通用**） | 不消耗房卡 |
| 游戏金币 | 禁止带入 | 消耗本游戏金币 |
| 平台抽成 | 无 | 5% 抽水 |
| 玩家来源 | 俱乐部 / 好友 | 陌生人匹配 |
| 跨游戏 | 房卡可开任意游戏房间 | 金币仅本游戏有效 |

---

## 5. 防套利规则

| 规则 | 说明 |
| :--- | :--- |
| 房卡场与记分场隔离 | 账号统计分开 |
| 游戏金币隔离 | 不同游戏金币不可互兑、不可转移 |
| 同 IP / 设备限制 | 详见 [risk/anti-fraud.md](../../risk/anti-fraud.md) |
| 俱乐部房禁金币 | 房卡场不可带入游戏金币 |
| 游戏金币不可换房卡 | 单向经济 |

详见 [matching-bankruptcy.md](matching-bankruptcy.md)。

---

## 6. 记分场盈利测算（示例）

**假设：** 单游戏（打乌龟），月活 1 万，30% 进记分场，日均 3 局。参数见 [games/dawugui/ops-hooks.md](../../games/dawugui/ops-hooks.md)。

| 指标 | 结果 |
| :--- | :--- |
| 日局数 | ~9,000 局 |
| 月抽水（人民币） | ~4,000 元（单游戏） |

*多游戏需分游戏统计。完整 KPI 见 [platform/metrics.md](../../platform/metrics.md)。*
