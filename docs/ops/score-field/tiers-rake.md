# 记分场抽水框架（通用）

> 官方运营 — 抽水模型与结算逻辑。与具体游戏无关。  
> **各游戏金币名称、场次、底分、准入**见 [games/{id}/ops-hooks.md](../../games/README.md)。

---

## 1. 通用结算公式

```
玩家游戏金币变动 = 规则积分 × 底分
平台抽水         = 赢家净赢 × 5%
```

- **规则积分**：由各游戏规则 PRD 与 ops-hooks 定义
- **底分 / 准入**：由各游戏 ops-hooks 中的场次表定义
- **游戏金币**：各游戏独立币种，不可跨游戏使用

---

## 2. 抽水模型

**推荐：赢家净赢 × 5%**

| 参数 | 值 | 说明 |
| :--- | :--- | :--- |
| 抽水率 | 5% | 行业常见 3~8%，全平台统一 |
| 抽水对象 | 赢家净赢 | 仅对正值抽水 |
| 特殊结算局 | 同上 | 不对代付部分重复抽水（如打乌龟包牌局） |

**可选：固定桌费**

```
桌费 = 底分 × 0.5
```

每局开局预扣，与人数无关。可与抽水二选一，或低场次桌费、高场次抽水。

---

## 3. 包牌类特殊结算（框架）

适用于存在「一方代付多方可失分」机制的游戏（如打乌龟包牌）：

- 代付方承担其他输家合计积分 × 底分
- 被代付方本局扣分为 0
- 平台仍只对赢家净赢部分抽水

具体触发条件与积分算法见各游戏 ops-hooks（如 [dawugui/ops-hooks.md](../../games/dawugui/ops-hooks.md) §3、§7）。
