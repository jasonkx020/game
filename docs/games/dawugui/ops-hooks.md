# 打乌龟 — 运营挂点

> 游戏层文档。规则 PRD 见 [dawugui.md](../../dawugui.md)。  
> **龟币**绑定本游戏；**房卡**绑定平台（全游戏通用，见 [platform/economy-base.md](../../platform/economy-base.md)）。

---

## 1. 玩法特征与运营策略

| 玩法特征 | 规则要点（dawugui.md） | 运营策略 |
| :--- | :--- | :--- |
| **零和博弈** | 1 赢家 + 多输家，积分守恒（§1、§9） | 适合记分场抽水，经济系统天然封闭 |
| **高波动** | 小王 5 分、大王 10 分，单局分差可达 30+（§9.1） | 设置底分/准入门槛，控制破产风险 |
| **包牌机制** | 上家放走报单者，承担所有其他输家总分（§7） | 名场面传播；包牌王排行榜 |
| **报单/作废** | 未报单跑牌则整局作废（§8） | 【产品需求】系统强制报单 |
| **3~5 人** | 人数弹性大（§1） | 俱乐部可按 4 人桌稳定组局 |

## 2. 游戏金币经济（龟币）

| 项目 | 配置 |
| :--- | :--- |
| **币种名称** | 打乌龟金币（简称「龟币」） |
| **绑定范围** | 仅打乌龟记分场；不可用于其他游戏 |
| **充值比例** | 1 元 = 10,000 龟币 |
| **不可提现** | 单向充值，不可兑换人民币或房卡 |

### 2.1 记分场场次分级

将 dawugui.md §9 积分映射为龟币底分：

| 场次 | 底分 | 准入龟币 | 单局预估波动 | 目标用户 |
| :--- | :--- | :--- | :--- | :--- |
| 新手场 | 100 | 5,000 | ±500 ~ 2,000 | 新注册用户 |
| 初级场 | 500 | 25,000 | ±2,500 ~ 10,000 | 稳定活跃用户 |
| 中级场 | 2,000 | 100,000 | ±10,000 ~ 40,000 | 高活跃付费用户 |
| 高级场 | 10,000 | 500,000 | ±50,000 ~ 200,000 | 高净值用户 |

### 2.2 结算公式

```
玩家龟币变动 = 规则积分 × 底分
平台抽水   = 赢家净赢 × 5%（框架见 ops/score-field/tiers-rake.md）
```

**包牌局：** 上家承担其他输家合计总分 × 底分；平台仍只对赢家净赢抽水。

### 2.3 产出与消耗

**产出（克制）：**

| 来源 | 数量 | 说明 |
| :--- | :--- | :--- |
| 注册礼包 | 5,000 龟币 + 5 房卡 | 房卡为平台通用；一次性，约 3~5 局新手场 |
| 每日签到 | 200 龟币 | VIP 翻倍 400 龟币 |
| 代理活动补贴 | 有上限 | 需满足流水要求 |

**消耗（核心）：** 局内输赢、道具/表情、破产救济、锦标赛报名费

**平衡目标：** 日产出 < 日消耗 70%，其余靠充值补充。

### 2.4 结算示例（4 人，初级场底分 500）

| 玩家 | 剩余手牌 | 规则积分 | 龟币变动 |
| :--- | :--- | :--- | :--- |
| A（赢家） | 0 | — | +7,500（三输家合计 15 分） |
| B | 3 张普通 + 1 小王 | 4 | -2,000 |
| C | 5 张普通 | 5 | -2,500 |
| D | 2 张普通 + 1 大王 | 12 | -6,000 |
| 平台抽水 | — | — | 375（7,500 × 5%） |

## 3. 积分算法

摘自 dawugui.md §9：

| 牌型 | 分值 |
| :--- | :--- |
| 普通牌（3 ~ 2, A, K...） | 1 分 / 张 |
| 小王 | 5 分 / 张 |
| 大王 | 10 分 / 张 |

**正常结算：** `输家扣分 = (普通牌数 × 1) + (小王数 × 5) + (大王数 × 10)`

**包牌结算（§7.1）：** 上家承担其他输家合计总分；其他输家扣分为 0。

## 6. 报单运营说明

**规则摘要（§6）：**
- 触发：打出一手牌后，手牌恰好变为 1 张
- 动作：立即宣告「报单」

**运营策略：**
- 【产品需求】系统自动报单
- 【运营动作】新手教程说明报单与包牌风险
- 【运营动作】客服话术：「系统已自动报单，无需手动操作」

---

## 7. 包牌运营说明

**规则摘要（§7）：**
- 前提：玩家 A 已报单，只剩 1 张
- 触发：上家出单张，A 压住并跑掉
- 结果：上家承担所有其他输家总分

**运营策略：**
- 【运营动作】推送「名场面」分享卡片
- 【运营动作】包牌王排行榜（见 [score-field/activities.md](../../ops/score-field/activities.md)）
- 【运营动作】俱乐部内包牌计入 MVP 积分
