# 记分场匹配与破产保护

> 官方运营 — 匹配与风险保护框架。具体准入数值见各游戏 ops-hooks。

---

## 1. 匹配规则

| 规则 | 说明 |
| :--- | :--- |
| 匹配方式 | 平台官方快速匹配，按**游戏** + 场次 + 金币区间 |
| 游戏隔离 | 打乌龟记分场仅匹配打乌龟玩家，不与其他游戏混场 |
| 人数 | 按各游戏规则 PRD（如打乌龟 3~5 人） |
| 同桌限制 | 遵循 [risk/anti-fraud.md](../../risk/anti-fraud.md) |

---

## 2. 场次准入

- 玩家**本游戏金币** ≥ 该游戏 ops-hooks 中定义的准入门槛，方可进入
- 不匹配时提示「金币不足，请充值或选择更低场次」

**准入数值示例（打乌龟）：** 见 [games/dawugui/ops-hooks.md](../../games/dawugui/ops-hooks.md) §2.1 场次表。

新游戏接入时在 ops-hooks 中定义各自场次与准入，不在此文档硬编码。

---

## 3. 破产保护

| 条件 | 系统动作 |
| :--- | :--- |
| 游戏金币 < 准入 × 30% | 弹出「破产保护」提示 |
| 破产保护触发 | 引导充值或降级至更低场次 |
| 连续 3 次破产 | 推送新手场体验包（金额与金币数见该游戏 ops-hooks） |

【运营动作】破产保护弹窗 A/B 测试：充值引导 vs 降级引导。

**打乌龟示例：** 6 元 = 10,000 龟币体验包，见 [dawugui/ops-hooks.md](../../games/dawugui/ops-hooks.md)。

---

## 4. 防套利（记分场侧）

| 规则 | 说明 |
| :--- | :--- |
| 与房卡场隔离 | 房卡与游戏金币不可互相转换 |
| 游戏金币隔离 | 不同游戏金币不可互兑 |
| 俱乐部房禁金币 | 房卡场不可带入游戏金币 |

房卡场规则见 [room-card/overview.md](../room-card/overview.md)。

---

## 5. 合规

游戏金币单向充值、不可提现、不可跨游戏转移，详见 [risk/compliance.md](../../risk/compliance.md)。

异常输赢复核详见 [risk/anti-fraud.md](../../risk/anti-fraud.md)。
