# 平台经济基座（房卡）

> **房卡**绑定平台，全游戏通用。游戏金币在各游戏 `ops-hooks.md` 中独立定义。  
> 记分场运营框架见 [ops/score-field/tiers-rake.md](../ops/score-field/tiers-rake.md)。

---

## 1. 货币绑定模型

| 货币 | 绑定层级 |  scope | 说明 |
| :--- | :--- | :--- | :--- |
| **房卡** | 平台 | 全游戏通用 | 创建/续费房间；各游戏可配置不同消耗数量 |
| **游戏金币** | 游戏 | 单游戏独立 | 记分场结算；不同游戏币种互不通用 |

**跨游戏货币规则：**

| 规则 | 说明 |
| :--- | :--- |
| 房卡跨游戏通用 | 同一账号房卡可用于任意已接入游戏开房 |
| 游戏金币隔离 | 打乌龟金币不可用于其他游戏，反之亦然 |
| 金币不可换房卡 | 单向经济，防止套利 |
| 游戏金币不可互兑 | 不同游戏金币不可转换 |
| 房卡不可局内输赢 | 房卡仅用于开房，不参与对局结算 |

各游戏金币名称、充值比例、场次分级见 [games/{id}/ops-hooks.md](../games/README.md)。

---

## 2. 房卡定义

| 项目 | 说明 |
| :--- | :--- |
| **别名** | 钻石 |
| **用途** | 创建/续费房间（房卡场，全游戏通用） |
| **获取方式** | 购买房卡包、俱乐部月卡、活动赠送 |
| **限制** | 不可在局内输赢；不可玩家间交易 |

房卡是 Phase 1 核心收入来源，主卖给代理与俱乐部管理员（[房卡场运营](../ops/room-card/overview.md)）。

## 3. 房卡定价包

| 产品 ID | 内容 | 零售价 | 平台毛利估算 |
| :--- | :--- | :--- | :--- |
| rc_10 | 10 张房卡 | 6 元 | ~85%（扣除支付 3%、代理 30~40%） |
| rc_50 | 50 张房卡 | 28 元 | ~85% |
| rc_200 | 200 张房卡 | 98 元 | ~85% |

**俱乐部月卡：** 198 元/月，含 100 房卡 + 管理后台

## 4. 房卡消耗规则（平台默认）

当游戏未在 ops-hooks 中覆盖时，使用以下默认值：

| 玩家人数 | 每局消耗房卡 |
| :--- | :--- |
| 3 人 | 2 张 |
| 4 人 | 2 张 |
| 5 人 | 3 张 |

**俱乐部自建桌：** 管理员预扣房卡池，成员免费进房。

各游戏可在 [games/{id}/ops-hooks.md](../games/README.md) 中覆盖消耗规则（如不同游戏、不同人数对应不同房卡数）。

## 5. 代理分成基线

| 规则项 | 参数 | 说明 |
| :--- | :--- | :--- |
| 一级代理供货价 | 零售价的 60% | 例：6 元房卡包，供货价 3.6 元 |
| 一级代理 GMV 返点 | 5~10% | 按区域月度 GMV 阶梯返点 |
| 俱乐部管理员返点 | 3~5% | 仅房卡 GMV，不含记分场 |
| 代理链总分成上限 | ≤ 45% | 确保平台每 6 元房卡包净赚 ≥ 2.5 元 |
| 支付通道费 | ~3% | 从平台侧扣除 |

详细代理体系见 [ops/room-card/agent-club.md](../ops/room-card/agent-club.md)。

**分成计算示例（6 元房卡包，代理链分成 35%）：**

```
零售价：6.00 元
代理分成：6.00 × 35% = 2.10 元
支付费用：6.00 × 3%  = 0.18 元
平台净利润：6.00 - 2.10 - 0.18 = 3.72 元（> 2.5 元目标）
```
