# 房卡场运营概览（地推）

> 房卡场由 **地推 / 代理 / 俱乐部** 体系运营，平台不直接匹配玩家。  
> 经济规则见 [platform/economy-base.md](../../platform/economy-base.md)；记分场见 [score-field/overview.md](../score-field/overview.md)（独立业务线）。

---

## 1. 定位

| 项目 | 说明 |
| :--- | :--- |
| **运营主体** | 地推团队、一级/二级代理、俱乐部管理员 |
| **场域性质** | 俱乐部 / 好友私密组局 |
| **变现方式** | 房卡销售、俱乐部月卡 |
| **目标用户** | 熟人社交圈、代理线下用户 |
| **平台抽成** | 无（纯房卡收入） |

---

## 2. Phase 1 阶段目标

**时间：** 0 ~ 3 月  
**核心目标：** 快速现金流，验证玩法与留存

| 指标 | 目标值 |
| :--- | :--- |
| 签约种子代理 | 10 个 |
| 活跃俱乐部 | 50 个 |
| 月 GMV | 3 万+ |
| 平台净利率 | > 30% |

---

## 3. MVP 盈利闭环

```
房卡购买 → 创建房间 → 消耗房卡 → 代理补货 → 循环
```

**闭环验证清单（8 周内）：**

1. 玩家可通过代理/平台购买房卡
2. 房主消耗房卡创建房间，成员免费加入
3. 每局结束自动扣减房卡
4. 俱乐部管理员可代充房卡池
5. 后台可见 GMV、房卡消耗率、代理 ROI

## 5. 无效局与房卡规则

| 场景 | 房卡处理 | 理由 |
| :--- | :--- | :--- |
| 正常结束 | 已消耗，不退 | — |
| 未报单作废 | 不退 | 房卡已在开局时扣除 |
| 玩家中途退出 | 不退 | 防止恶意刷房卡 |

【产品需求】系统强制报单，使无效局趋近于 0（详见 [games/dawugui/ops-hooks.md](../../games/dawugui/ops-hooks.md)）。

## 6. 与记分场的边界

| 规则 | 房卡场 | 记分场 |
| :--- | :--- | :--- |
| 运营主体 | 地推 / 代理 | 平台官方 |
| 房卡 | 消耗**平台通用房卡**（各游戏消耗数量见 ops-hooks） | 不消耗房卡 |
| 游戏金币 | 禁止带入 | 消耗**该游戏独立金币** |
| 平台抽成 | 无 | 5% 抽水 |
| 玩家来源 | 俱乐部 / 好友 | 陌生人匹配 |
| 跨游戏 | 同一房卡可开任意已接入游戏房间 | 金币仅本游戏有效 |

防套利与隔离规则详见 [score-field/overview.md](../score-field/overview.md)。
